## 2. 前端架构（统一 Monorepo）

前端不分师生两套仓库：`apps/web`（师生合一）与 `apps/admin`（运营后台）共用 `packages/*`，通过角色与路由区分能力。

### 2.1 Monorepo 分层

```
apps/                          packages/
┌─────────────┐                ┌─────────────┐
│ web         │───────────────►│ types       │  领域模型
│ 师生前台    │───────────────►│ api-client  │  HTTP 端点
└─────────────┘                │ ui          │  基础组件
┌─────────────┐                │ shared      │  工具常量
│ admin       │───────────────►│ config      │  工程配置
│ 运营后台    │                └─────────────┘
└─────────────┘
```

**约定（与 README 一致）**

- `app/` 只做路由薄壳，业务沉到 `features/<域>/`
- 数据请求：TanStack Query + `packages/api-client`
- 客户端状态：Zustand（如 `stores/user.ts`）
- 样式：Tailwind CSS + `@tsz/ui`

## 3. 后端架构（Java + Go）

### 3.1 语言分工原则

|              | Java 基础服务                                        | Go 高性能模块                                  |
| ------------ | ---------------------------------------------------- | ---------------------------------------------- |
| **定位**     | 业务主链路、事务一致性、运营后台                     | CPU/IO 密集、高并发、低延迟工具                |
| **典型场景** | 用户注册、词库 CRUD、审批流、班级任务配置            | 智能组题、实时判分、批量统计、模糊检索         |
| **框架建议** | Spring Boot 3 · MyBatis-Plus / JPA · PostgreSQL 驱动 | Gin / Fiber · pgx · gRPC（对内）               |
| **对外暴露** | 大部分 REST API（`/api/*`）                          | 练习/统计/匹配专用 API（`/api/practice/*` 等） |

### 3.2 Java 基础服务（按业务域分包）

建议仓库路径：`services/java/tsz-api/`（模块化单体起步，按域分包）。

```
Java · tsz-api
├── auth           ① 账号与身份
├── lexicon        ② 标准智能词库
├── user-lexicon   ③ 非标准智能词库  ──► 依赖 lexicon
├── sentence       ④ 多维例句        ──► 依赖 lexicon, user-lexicon
├── wordlist       ⑤ 词表与学习集
├── task           ⑦ 教学任务        ──► 依赖 class, wordlist
├── class          ⑧ 班级
├── coin           ⑩ 天生币积分
├── social         ⑪ 邀新与互动
└── admin          ⑫ 平台运营后台
```

| 服务域         | 业务模块 | 职责                                          |
| -------------- | -------- | --------------------------------------------- |
| `auth`         | ①        | 注册登录、JWT、角色、老师实人认证             |
| `lexicon`      | ②        | 标准词条、难度、COCA、标准词表、TTS 任务投递  |
| `user-lexicon` | ③        | 用户词库 CRUD、审批流                         |
| `sentence`     | ④        | 例句录入、成分配置、译文管理；匹配请求转发 Go |
| `wordlist`     | ⑤        | 词表、学习集、收藏、公开/私密                 |
| `task`         | ⑦        | 长期/每日任务配置、对象绑定、进度元数据       |
| `class`        | ⑧        | 班级、邀请、加入审批、班级码                  |
| `coin`         | ⑩        | 积分账本、规则、发放与消耗                    |
| `social`       | ⑪        | 邀请码、点赞、评论                            |
| `admin`        | ⑫        | 运营配置、专职角色、批量导入                  |

### 3.3 Go 高性能模块

建议仓库路径：`services/go/`，各模块独立二进制，便于水平扩展。

```
Go 服务
├── practice-engine   ⑥ 学习与练习（组题·判分）
│       │
│       └──► analytics  ⑨ 学情数据中心
├── lexicon-search    ④ 成分用词模糊匹配
├── scheduler         ⑦ 每日 4:00 任务刷新 ──► practice-engine
└── worker            ②④ TTS 回调 · 异步流水线
```

| 服务              | 业务模块 | 职责                                       | 关键接口（示例）                                   |
| ----------------- | -------- | ------------------------------------------ | -------------------------------------------------- |
| `practice-engine` | ⑥        | 按题型组题、干扰项生成、作答判分、练习会话 | `POST /practice/sessions`、`POST /practice/submit` |
| `analytics`       | ⑨        | 进度聚合、正确率、学情预警、Top10 薄弱词   | `GET /stats/class/:id`、`GET /stats/student/:id`   |
| `lexicon-search`  | ④        | 例句「开始匹配」模糊检索、词形候选         | `POST /search/lexicon/match`                       |
| `scheduler`       | ⑦        | 每天 04:00 刷新每日任务进度、触发组题预热  | 内部 cron，无公网暴露                              |
| `worker`          | ②④       | TTS 回调处理、音频落 OSS、练习事件异步消费 | 队列消费，无公网暴露                               |

### 3.4 Java ↔ Go 协作

```
【例句成分匹配】
前端 ──POST /sentences+匹配──► Java
Java ──gRPC MatchComponents──► Go lexicon-search
Go   ──候选词条列表──────────► Java ──► 前端（用户确认词义）

【创建每日任务】
前端 ──POST /tasks──► Java ──► PostgreSQL 持久化任务配置

【学生练习】
前端 ──POST /practice/sessions──► Go practice-engine
Go   ──读 PostgreSQL 词表·词库·例句──► 返回题目列表 ──► 前端
前端 ──POST /practice/submit──► Go practice-engine
Go   ──异步推送作答事件──► Go analytics ──► PostgreSQL 聚合学情

【查看学情】
前端 ──GET /tasks/:id──────────► Java（任务元数据）
前端 ──GET /stats/class/:id───► Go analytics（统计详情）
```

**协作约定**

- Java 持有业务真相源（用户、词库、任务、班级）；Go 读写练习/统计专用表，词库类表以只读为主。
- 对内通信用 **gRPC**（低延迟）或内部 HTTP；对外仍走 Gateway 统一域名。
- 练习相关请求由 Gateway 路由到 Go；鉴权 Token 由 Gateway 插件校验，或 Go 调 Java `GET /auth/introspect`。

### 3.6 出题引擎（Go · 模块 ⑥ 核心）


**出题规则摘要**

- 单词类：跟读 / 单选 / 听写 / 翻译 / 拼写
- 语法结构类：跟读 / 单选 / 听写 / 多选 / 拼写
- 上下文依赖词自动选题型；干扰项同类词生成、排除近义
- 例句按考查词义选取，加粗对应 BrE/AmE 词形

| 数据类型                         | 存储                                                   | 说明                               |

## 8. 演进路径

```
Phase 1                Phase 2                    Phase 3
Java 单体 API     ──►  Go 补齐 analytics      ──►  Go 水平扩容
+ Go practice-engine    scheduler · worker          练习·统计高峰
     │                       │                          │
     └───────────────────────┴──────────────────────────┘
                    前端切真实接口，逐步替换 mock
```

1. **Phase 1**：Java 承载 CRUD 主链路；Go 先落地 `practice-engine`，前端切真实接口。
2. **Phase 2**：Go 补齐 `analytics`、`lexicon-search`、`scheduler`、`worker`。
3. **Phase 3**：练习与统计流量增大时，对 Go 服务组独立扩容；Java 保持业务配置类负载。
