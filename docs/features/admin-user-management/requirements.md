# 后台「用户管理」重构（C 端用户 + 管理员管理） 需求评估

## 背景与目标

管理后台侧栏当前的「用户管理」分组下挂着两个 disabled 的占位项——「教师管理」和「学生管理」，
路由 `/users` 指向一个只有标题的空壳页 `Users.tsx`，尚无任何真实功能。

产品原型把「用户管理」拆成「教师管理 / 学生管理」两块，但**教师和学生在后端是同一套账号体系**
（师生合一，`roles: ("student" | "teacher")[]`，一个人可同时是学生和教师），拆两个页面既不贴合数据
模型、也会造成重复。因此本次**合并为一个「用户管理」页**，用角色筛选（全部 / 老师 / 学生）区分，
管的是 **C 端（web 学员/教师）用户**。

与此同时，后台自身的**管理员账号**（`admin` / `super_admin`，与 C 端完全独立的身份体系）也需要一个
管理入口。后端这套接口（建号 / 列表 / 启禁用 / 重置密码）已完整实现并冻结契约，但前端一直没有落地
页面。本次一并补上，命名为「管理员管理」。

**成功长什么样**：后台侧栏「用户管理」分组下有两个可点击的 section——「用户管理」（C 端用户）与
「管理员管理」（后台管理员），各自有真实可用的列表/筛选/操作，替换掉现有的死链占位与空壳页。

## 目标端

**admin**（管理后台，Vite + React Router + TanStack Query + antd v6）。不涉及 web、不涉及 `@tsz/ui`。
类型与请求层改动落在 `@tsz/types` / `@tsz/api-client`（两端共享包，但本次只有 admin 消费）。

## 用户故事 / 使用场景

### Section A —— 用户管理（C 端用户，所有 admin 可见）

- 作为**后台运营/管理员**，我在「用户管理」页看到一张 C 端用户列表（师生合一），
  想按**昵称 / 手机号 / 邮箱 / 注册时间**搜索、按**全部 / 老师 / 学生**切换筛选，
  快速定位到某个用户，查看其账号信息（昵称、角色、绑定手机/邮箱、天生币余额、状态、注册时间）。
- 作为管理员，列表默认按注册时间倒序（最新用户在最前），可翻页。
- 作为管理员，点某一行可查看该用户的**详情**（本次为只读展示，聚合列表已返回的字段）。

### Section B —— 管理员管理（后台管理员账号）

- 作为**超级管理员**（`super_admin`），我在「管理员管理」页看到全部管理员账号列表，
  可按 level / 关键字过滤、翻页。
- 作为超管，我能**新建管理员**（填手机号、初始密码、昵称、level），成功后新账号出现在列表。
- 作为超管，我能**启用/禁用**某个管理员（禁用后该账号无法登录/刷新，≤1 个 access TTL 生效）。
- 作为超管，我能**重置某个普通管理员（level=admin）的密码**，系统返回一次性临时密码，
  我复制后线下转交，对方下次登录被强制改密。
- 作为**普通管理员**（`admin`），我在侧栏**看不到「管理员管理」入口**（前端按 `level` 直接隐藏，
  不依赖后端 403 兜底）。

## 功能范围

> **交付策略（本次核心决定）**：**契约已就绪的部分对接真实接口；契约缺失的部分先前端 mock**，
> 让页面按产品原型呈现完整形态，并为每个缺口建**待办事项**（面向后端的清单
> `backend-todos.md` + 代码里 `// TODO(backend): …` 注释）。接口/字段补齐后，把 mock 数据层替换为真实
> `api.*` 调用即可，mock 代码集中隔离、易删。

### 做什么（本次范围内）

- **侧栏改造**：把「用户管理」分组下的 `教师管理 / 学生管理` 两个 disabled 占位，替换为两个真实叶子
  —— `用户管理`（→ `/users`）、`管理员管理`（→ `/admins`，仅 `super_admin` 可见）。
- **用户管理页（C 端）——按原型呈现完整表格，真实 + mock 混合**：
  - 顶部搜索区：用户昵称、手机号、邮箱号码、注册时间（日期）四个筛选 + 搜索 / 重置。
    （昵称/手机/邮箱走真实 `q`；注册时间筛选后端暂无参数 → **mock 客户端过滤** + TODO。）
  - 角色 tab：全部 / 老师 / 学生（映射后端 `role` 查询参数：全部=不传、老师=`teacher`、学生=`student`）。
  - 列表表格：契约已有字段（用户 ID、昵称、角色、绑定手机、绑定邮箱、天生币余额、状态、注册时间）走
    真实契约；原型多出的**头像、等级（A1/A2…）、更新时间**列 → **mock 数据补齐** + TODO；分页。
  - 用户详情查看（只读抽屉）：真实字段 + mock 补充字段展示。
  - 行操作：
    - **编辑 / 删除 / 启禁用（用户）**：契约缺失 → **mock 本地行为**（可演示）+ TODO，接口就绪替换。
    - **天生币管理 / 等级管理 / 方言管理**：三者是**独立功能模块**（天生币侧栏另有分组），本次**不 mock 完整
      弹窗**——渲染操作入口，点击给「功能待接入」占位提示 + TODO，待各模块单独立项。
- **管理员管理页（后台管理员，完整，接真实接口）**：
  - 列表：`GET /admin/admins`，按 level / 关键字过滤 + 分页。
  - 新建管理员：`POST /admin/admins`（表单校验对齐契约：密码 ≥12、昵称禁 `< >` 及控制字符等）。
  - 启用/禁用：`PATCH /admin/admins/{id}/status`（含「禁用最后一个 super_admin 被 409 拒绝」的错误提示）。
  - 重置密码：`POST /admin/admins/{id}/reset-password`，弹窗一次性展示临时密码 + 复制。
  - 入口与页面均按 `super_admin` 门控。
- **类型与请求层**：
  - **admin wire 类型统一收敛到 `@tsz/types`**——把现散在 `@tsz/api-client` 的 `Admin` / `AdminLevel` /
    `AdminStatus` / `AdminProfile` / `AdminAuthResponse` **上提到 `@tsz/types`**，并新增 `AdminUser` /
    `AdminUserListResponse` / `CreateAdminInput` / `AdminListResponse` 等；api-client 与 shared 改从
    `@tsz/types` 引用（详见 design「Admin 类型归属」决定）。
  - `@tsz/api-client` 的 `createAdminEndpoints` 补 `users.list`、`admins.{list,create,setStatus,resetPassword}`。

### 明确不做什么（划走边界）

- **不做**天生币管理 / 等级管理 / 方言管理的完整功能与 mock 弹窗——它们是独立模块，本次仅留操作入口占位
  - TODO；有各自接口后单独立项。
- **不接后端**产品原型里 C 端用户的写操作（编辑/删除/启禁用）——**本次仅 mock 演示**，真实生效待后端接口。
- **不做**班级管理、审核等其它侧栏分组——不在本需求内。
- **不碰** web 端、不碰 `tsz-go` 后端实现（缺口只写建议清单，由用户转达）。

## 约束与边界

- **两套身份体系隔离**：C 端用户走 `/api/v1/admin/users`（用 admin token），管理员账号走
  `/api/v1/admin/admins`；二者都是后台接口、都带 admin `Authorization` + admin refresh cookie，
  与 web 学员/教师端账号无关。
- **权限门控**：「管理员管理」全部接口要求 `super_admin`，普通 admin 调用得 403 `super admin required`。
  前端按 `useAuthStore` 里的 `profile.level` **直接隐藏入口 + 页面守卫**，不靠后端兜底。
- **列表分页契约**：`?page=1&page_size=20`（`page_size` 上限 100），响应
  `{ items, page: { page, page_size, total } }`。
- **错误格式**：统一 `{ "error": "..." }`。需处理的典型值：建号 `409 phone already registered`、
  禁用最后超管 `409`、重置超管 `403 cannot reset a super admin`、通用 `401`（走后台 401 刷新重试）。
- **后端就绪度**：管理员账号接口**已实现**，可真实联调；C 端 `GET /admin/users` 契约冻结但后端**可能
  尚未实现**（文档 §9 标 B 段「先按契约 mock」），联调前用 mock/stub 保 UI 与状态流。
- **antd v6 硬约定**：`Form.List` 用 `field.key`、`noStyle` 的 `Form.Item` 上 `style` 无效、
  `Alert` 用 `title` 非 `message`、两字按钮插空格；jsdom 测试需 `matchMedia`/`ResizeObserver` 垫片、
  大表格测试避免 `getByRole`。禁止引入 tailwind / `@tsz/ui`。

## 验收标准

- [ ] 侧栏「用户管理」分组下显示两个**可点击**叶子：`用户管理`、`管理员管理`；不再有 disabled 死链。
- [ ] 普通 admin 登录时，侧栏**看不到**「管理员管理」；super_admin 登录时**能看到**。
- [ ] `/users` 页按原型呈现完整表格（含头像/等级/更新时间 mock 列），昵称/手机/邮箱/注册时间筛选与
      全部/老师/学生 tab 均生效，能翻页；契约字段走真实、缺失字段/筛选走 mock。
- [ ] `/users` 列表按注册时间倒序，点行能看到详情抽屉。
- [ ] `/users` 行操作：编辑/删除/启禁用能 mock 演示（本地生效）；天生币/等级/方言点击给「功能待接入」占位。
- [ ] 直接访问 `/admins`（普通 admin）被守卫拦截（重定向/无权限提示）；super_admin 正常进入。
- [ ] `/admins` 页能列出管理员、能新建（含手机号重复 409 提示）、能启禁用、能重置密码（临时密码可复制）。
- [ ] 禁用最后一个 super_admin 时给出 409 的明确错误提示，不误导为成功。
- [ ] 所有缺口在 `backend-todos.md` 有对应条目，代码里有 `// TODO(backend): …` 锚点。
- [ ] mock 数据层集中隔离（独立文件 + 切换开关），替换为真实 API 时改动面小。
- [ ] `pnpm typecheck` / `pnpm lint` / `pnpm test:cov`（admin 90% 门槛）全绿。

## 开放问题（多数已拍板，留待后端）

- **已定**：缺接口的功能**先前端 mock + 建待办**，用户转达后端；接口就绪替换真实调用。
- **已定**：admin wire 类型**上提 `@tsz/types`** 统一收敛（见 design）。
- **待后端**：`AdminUser` 补 `avatar_url` / 语言等级 / `updated_at`；`GET /admin/users` 补注册时间筛选参数；
  补用户详情、用户启禁用接口；天生币/等级/方言各自立契约。—— 均在 `backend-todos.md` 列清，由用户转达后端。
- **待产品/后端**：C 端用户详情页最终要展示哪些信息（学习记录/订单等）——本次先只读展示已有字段。
